# Support

Use the support path that matches the kind of help you need in Prompthon Agentic Labs.

## Before opening anything

Start with:

- `README.md`
- `CONTRIBUTING.md`
- `contributor-kit/index.mdx`
- `contributor-kit/contribution-workflow.mdx`
- `contributor-kit/github-issue-guide.mdx`

Those documents define the repository structure, contribution types, templates,
and review expectations.

## Quick routing

| Need | Best first place |
| --- | --- |
| You are reading and cannot find support for a confusing point | Discord `#general-chats` |
| You want to suggest a handbook improvement but the shape is still early | Discord `#handbook-contributors` |
| You found a concrete bug, broken link, or factual problem | GitHub `Bug Or Broken Surface Report` |
| You want to propose new handbook content | GitHub `Content Proposal` |
| You want a repository feature or process change | GitHub `Repository Feature Or Process Proposal` |
| You want to contribute to an existing task | GitHub issue comment, after checking `#good-first-issues-feed` |
| You need PR review coordination | The PR thread first, then Discord `#code-review` for lightweight coordination |

For the full sequence from issue to pull request, use
`contributor-kit/contribution-workflow.mdx`.

## Open an issue when

Open a GitHub issue if you have:

- a concrete bug or broken path
- a lab content proposal
- a source-project proposal
- a practitioner skill package proposal
- a repository feature or process proposal

Use the repository's issue forms so the request lands in the right shape for
review.

## Use community discussion when

Use the Prompthon IO Discord for:

- early questions before you know whether something should become an issue
- help choosing the right contribution surface
- general discussion that is not yet a scoped work item

Use `#general-chats` for reader questions, `#handbook-contributors` for
contribution routing, `#good-first-issues-feed` when looking for work,
`#code-review` for lightweight PR coordination, and `#ai-trends` for early
ecosystem discussion.

Discord: https://discord.gg/sDE2HhGTg4

## Security reports

Do not file public issues for vulnerabilities or sensitive configuration
problems. Follow `SECURITY.md` instead.
